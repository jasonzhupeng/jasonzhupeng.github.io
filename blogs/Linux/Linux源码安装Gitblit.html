    Linux源码安装Gitblit

    jason 6/2/2023 Linux

    Git是一款优秀的代码管理工具，拥有分布式的代码版本控制，上传下载速度快效率高，功能十分强大，是开发者首选的代码管理工具；Gitblit是一个基于git的远程代码仓库，简便好用，搭配git使用事半功倍，可以提高代码产出效率，此文通过详细步骤讲解如何在Linux上安装Gitblit搭建自己的远程仓库

    # 安装java运行环境jre

    由于gitblit是java开发的，依赖java环境，所以需要先安装jre。官方下载地址：https://www.openlogic.com/openjdk-downloads (opens new window) 直接选择jre，无需安装jdk 或者

    cd /usr/local/src
wget https://builds.openlogic.com/downloadJDK/openlogic-openjdk-jre/8u372-b07/openlogic-openjdk-jre-8u372-b07-linux-x64.tar.gz

    解压安装包，重命名

    cd /usr/local/src
tar -xzvf openlogic-openjdk-jre-8u372-b07-linux-x64.tar.gz
mv openlogic-openjdk-jre-8u372-b07-linux-x64 java-jre

    配置环境变量

    vim /etc/profile

# 增加配置
export JRE_HOME=/usr/local/src/java-jre
export PATH=$JRE_HOME/bin:$PATH

source /etc/profile

    # 下载解压Gitblit源码安装包

    Gitblit官方下载地址：http://www.gitblit.com/ (opens new window)，目前最新版本是1.9.3，将源码包直接下载到服务器src目录

    cd /usr/local/src
wget https://github.com/gitblit/gitblit/releases/download/v1.9.3/gitblit-1.9.3.tar.gz
tar -xzvf gitblit-1.9.3.tar.gz

    # Gitblit配置

    编辑默认配置文件defaults.properties，修改如下参数

    vim /usr/local/src/gitblit-1.9.3/data/defaults.properties

 server.httpPort = 8999
 server.httpsProt = 8443
 server.shutdownPort = 8081
 git.sshPort = 29418
 server.httpsBindInterface = localhost
 #web.siteName = my develop Hub #站点名称
 #web.allowGravatar = false #是否显示用户头像
 #web.showBranchGraph = false #概况是否显示分支图

    编辑文件service-centos.sh，根据自己实际安装的路径修改参数，端口，和日志目录

    vim /usr/local/src/gitblit-1.9.3/service-centos.sh

GITBLIT_PATH=/usr/local/src/gitblit-1.9.3
GITBLIT_BASE_FOLDER=/usr/local/src/gitblit-1.9.3/data
GITBLIT_HTTP_PORT=8999
GITBLIT_HTTPS_PORT=8443
GITBLIT_LOG=/usr/local/src/gitblit-1.9.3/logs/gitblit.log

    创建Gitblit服务

    cd /usr/local/src/gitblit-1.9.3
cp ./service-centos.sh /etc/rc.d/init.d/gitblit
chkconfig --add gitblit

    最后验证服务是否能运行，输入命令

    systemctl start gitblit
systemctl status gitblit

    出现如下界面，安装运行成功

    # 配置Nginx访问端口

    最后再配置Nginx，通过外网可以访问gitblit的管理界面，在nginx.conf增加

    vim /usr/local/webserver/nginx/conf/nginx.conf

server {
    listen       443 ssl;
    server_name  repo.luckyzp.top;
    
    ssl_certificate      cert/9815538_repo.luckyzp.top.pem;
    ssl_certificate_key  cert/9815538_repo.luckyzp.top.key;

    ssl_session_timeout  5m;

    ssl_ciphers  HIGH:!aNULL:!MD5;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers  on;

    location / {
        proxy_pass https://localhost:8443/;
    }
}

    # 常见问题解决方式

    # 一.停止gitblit服务后，站点还能访问，实际服务还在后台运行

    解决方式为：修改service-centos.sh文件，修改stop相关代码






     





      stop)
    if [ -f $GITBLIT_PATH/gitblit.jar ];
      then
      echo $"Stopping gitblit server"
      cd $GITBLIT_PATH
      $JAVA "$GITBLIT_PATH/gitblit.jar:$GITBLIT_PATH/ext/*" com.gitblit.GitBlitServer --baseFolder $GITBLIT_BASE_FOLDER --stop > /dev/null &
      echo "."
      exit $RETVAL
    fi
  ;;

    重新创建服务，再重新打开，停止服务就能解决这个问题

    rm -f /etc/rc.d/init.d/gitblit
cp /usr/local/src/gitblit-1.9.3/service-centos.sh /etc/rc.d/init.d/gitblit
systemctl daemon-reload

    # 二.版本库地址显示localhost

    刚部署完成会发现一个问题，所有新建的版本库git地址都不显示IP地址或域名，而是显示localhost

    解决方式为：修改defaults.properties文件

    web.canonicalUrl = https://repo.luckyzp.top

    再重启服务，显示正常了
